# Windowed Training з Trace Aggregation

## Опис

Новий підхід до тренування моделі на датасетах з віконцями (windows), де кожен трейс розділений на множину вікон. Модель навчається на окремих вікнах, але фінальний prediction робиться на рівні трейсу через агрегацію MEAN.

## Структура датасету

```csv
trace_id,syscall_0,syscall_1,...,syscall_999,label
train_0,18,6,6,...,28,True
train_0,6,21,21,...,28,True
train_1,21,6,6,...,21,False
...
```

- **trace_id** - ідентифікатор трейсу (перша колонка)
- **syscall_0...syscall_999** - системні виклики у вікні (1000 syscalls)
- **label** - мітка класу (остання колонка): True/False або Malware/Benign

## Як працює

### 1. Тренування
Модель навчається на **окремих вікнах** як на незалежних зразках.

**ВАЖЛИВО:** KFold split робиться на рівні **ТРЕЙСІВ**, а не вікон!
- Train трейси: всі вікна йдуть в train
- Test трейси: всі вікна йдуть в test
- Жодного перетину між train та test трейсами

Це запобігає data leakage, коли модель бачить частину вікон трейсу під час тренування, а потім тестується на інших вікнах того ж трейсу.

### 2. Prediction
Для кожного вікна модель робить prediction (ймовірність класу).

### 3. Агрегація MEAN
Всі predictions для вікон одного трейсу **усереднюються**:
```python
trace_prediction = MEAN(window_predictions)
final_class = 1 if trace_prediction > 0.5 else 0
```

### 4. Метрики
Виводяться дві метрики:
- **Window-based Accuracy** - точність на рівні вікон
- **Trace-based Accuracy** - точність на рівні трейсів (після агрегації)

### Інтерпретація

- **Window-based Accuracy** - наскільки добре модель класифікує окремі вікна
- **Trace-based Accuracy** - наскільки добре модель класифікує цілі трейси після агрегації
- Зазвичай **Trace-based Accuracy вища**, оскільки агрегація зменшує вплив шумних predictions окремих вікон

## Переваги підходу

1. ✅ **Більш стабільні predictions** - агрегація зменшує вплив шуму
2. ✅ **Реалістична оцінка** - accuracy на рівні трейсів відповідає реальному використанню
3. ✅ **Гнучкість** - можна експериментувати з різними методами агрегації (MEAN, MAX, VOTING)
4. ✅ **Масштабованість** - можна обробляти трейси будь-якої довжини

## Порівняння з стандартним підходом

| Метрика | Стандартний | Windowed + Aggregation |
|---------|-------------|------------------------|
| Одиниця оцінки | Весь трейс | Вікно → Трейс |
| Довжина входу | Фіксована | Фіксована (вікно) |
| Обробка довгих трейсів | Обрізання/padding | Природна через вікна |
| Стабільність | Залежить від довжини | Висока через агрегацію |
